# PRD_SIMPLIFIED_FOR_CODEX

## 1. One-line Definition

《大馋猫：深夜占卜》 is an AI-assisted emotional food divination web game.

The player chooses a current emotional knot, receives a food riddle from the cat master, selects two ingredients from a late-night Hong Kong-style shop, sacrifices them, and receives a food result with cat-style wisdom.

---

## 2. Product Positioning

This is a short emotional interaction game, not a cooking simulator or management game.

Core experience:

```text
emotion
→ riddle
→ food ingredients
→ sacrifice
→ result wisdom
```

Design keywords:

- late night
- Hong Kong-style neon food stalls
- cats
- emotional healing
- slightly sarcastic but warm
- food metaphor
- lightweight puzzle-solving

---

## 3. V1 Baseline

V1 was a 0-shot single-page HTML prototype.

It validated the core loop:

```text
choose one of five emotional knots
→ read a riddle
→ switch shop / choose ingredients
→ select two ingredients
→ receive food and wisdom
```

V1 default emotional knots:

1. Deadline 任务过载
2. 此时此刻的嫉妒心
3. 对性格短板的怀疑
4. 报复性熬夜
5. 倒霉透顶求转机

V1 key lesson:

The five-entry emotional selection is simple and understandable.  
It should remain the default playable entry in V2/V3 unless explicitly redesigned.

---

## 4. Current V2 Playable-lite Status

V2 has transformed the prototype into a data-driven playable web game.

Implemented:

- CSV content source
- Runtime JSON generated by Node.js
- Frontend loads `content/runtime-data.json`
- Lightweight rainy opening cover
- `[ 推门而入 ]` entry
- Cat master intro
- Five V1 seed issues as default entry
- Two sacrifice slots
- Submit button
- Success / half-success / failure result overlays
- Random nonsense-slip failure
- Local validation scripts
- Release notes and README

Current V2 gameplay:

```text
rainy opening
→ cat master intro
→ five seed issue choices
→ riddle/shop/ingredients
→ sacrifice two ingredients
→ submit
→ success / half-success / failure
```

Important:

The full 40-issue dataset exists in runtime data, but it is not the default entry screen.

## 7. V3 Goal

V3 should not try to fully implement every 0424 feature at once.

V3 should become an immersive demo version based on stable V2.

V3 priorities:

1. Copy V2 as stable baseline.
2. Add asset folder system.
3. Add asset manifest.
4. Add image placeholder slots.
5. Add audio hook system.
6. Make five seed issues feel like mood cards.
7. Polish shop / ingredient / sacrifice visuals.
8. Add lightweight judgement transition.
9. Polish result card presentation.
10. Prepare for web deployment and final demo.

---

## 8. Recommended V3 Non-goals

Do not prioritize these first:

- Full 40-issue public card system
- Full collection book persistence
- All punishment interactions
- Full sprite animation system
- Real backend database
- Real LLM API integration
- Major framework migration

These can remain future expansion hooks.

---

## 9. Final Demo Focus

The final 4-minute demo should show:

```text
open web demo
→ rainy entrance
→ cat intro
→ choose emotional knot
→ read riddle
→ choose ingredients
→ submit sacrifice
→ show half-success or failure once
→ retry
→ show success food wisdom card
```

The demo should emphasize:

- concept originality
- emotional food metaphor
- AI-assisted development workflow
- stable web deployment
- fun and readable interaction
